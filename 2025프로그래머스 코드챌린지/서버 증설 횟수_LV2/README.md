# 문제 제목 
- 서버 증설 횟수

# 문제 링크
- https://school.programmers.co.kr/learn/courses/30/lessons/389479

# 문제 핵심
- 각 시간마다 "지금 운영 중인 서버 수"가 부족하면 그때 필요한 만큼만 바로 증설
player[i] / m
    •	2명, m=3 → 0대
	•	3명, m=3 → 1대
	•	4명, m=3 → 1대
	•	6명, m=3 → 2대

- 풀이순서
    1. 각 시간마다 필요한 서버 수계산
    2. 현재 운영 중인 서버 수 관리
    3. 부족하면 그만큼 증설
    4. 증설한 서버는 i+k 시간에 종료되도록 기록 (i = 현재시간)

# 제한 사항
- players = 시간대별 이용자수 = 24
- m = 서버 한대가 감당 할수 있는 최대 이용자수 
- k = 서버 한대가 운영 가능한 시간

# 문제 유형 
- 그리디 (매 순간 가장 좋아 보이는 선택을 하는 알고리즘)
- 시물레이션 (시간대별 해야하는 것을 그대로 코드로 따라가는 것)

# 문제 난이도
- Level 2
